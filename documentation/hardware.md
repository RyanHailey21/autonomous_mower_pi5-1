| Subsystem                           | Component                    |                                   Specifications | Project Role / Integration Notes                                                                                                                                                                       |
| ----------------------------------- | ---------------------------- | -----------------------------------------------: | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Compute                             | Raspberry Pi 5, 16GB         |                      2.4GHz quad-core, 3.3V GPIO | Main ROS2 computer. Runs Ubuntu Server 24.04, ROS2 Jazzy, SLAM/navigation, sensor drivers, serial bridge, logging, and high-level autonomy.                                                            |
| Microcontroller / Safety Logic      | Arduino Uno R4 Minima        |      Renesas RA4M1, USB-C serial, 5V logic board | Low-level controller. Should handle watchdog, E-stop inputs, motor enable logic, blade interlock, PWM outputs, encoder polling if needed, and fail-safe shutdown if Pi/ROS2 stops sending heartbeat.   |
| Logic Power, Pi Rail               | UCTRONICS U6112 #1           |                       24V to 5V, 5A, dual output | Dedicated 5V supply for Raspberry Pi 5. Ideally keep Pi power isolated from noisy motors and high-current loads.                                                                                       |
| Logic Power — Sensor/Accessory Rail | UCTRONICS U6112 #2           |                       24V to 5V, 5A, dual output | Separate 5V rail for sensors, Arduino, USB accessories, PWM-to-voltage modules, and logic-side electronics. Grounds must be common with the Pi and motor controllers.                                  |
| Main Energy                         | 24V 20Ah LiFePO4 Battery     |                            480Wh, integrated BMS | Primary mower battery. Feeds fuse box, motor controllers, blade motor controller, and 24V-to-5V buck converters.                                                                                       |
| Power Distribution                  | 6-Way PBT Fuse Box           |  100A capacity, M5 positive studs, M4 ground bus | Main 24V distribution. Use separate fused branches for left drive, right drive, blade controller, Pi buck converter, sensor buck converter, and auxiliary systems.                                     |
| Drivetrain Motors                   | LINIX 250W BLDC ×2           | 24V, 15A, sensored Hall feedback, gear reduction | Left and right wheel drive motors. Used for differential drive. Hall feedback can support speed estimation if exposed by the drivers or separately readable.                                           |
| Drive Motor Controllers             | ZS-X11H Drivers ×2           |  16A nominal, 0–5V analog speed command required | Controls the BLDC wheel motors. Needs a safe enable/disable strategy. Since it expects 0–5V analog, use PWM-to-voltage modules or a DAC stage from the Arduino. Test output voltage before connecting. |
| Drive Command Interface             | PWM-to-Voltage Bridge ×2     |                  PWM input to 0–5V analog output | Converts Arduino PWM commands into analog throttle signals for the ZS-X11H drivers. Should be tested with a multimeter and clamped/limited so it cannot command unexpected full speed.                 |
| Cutting Motor                       | 775 DC Motor                 |                            24V, about 10,000 RPM | Blade motor. Treat as hazardous. Should be integrated last after drive, sensing, E-stop, and software watchdogs are working.                                                                           |
| Cutting Blade                       | 20 cm Manganese Steel Blade  |                              20 cm cutting blade | Cutting element. Requires physical guarding, blade-enable interlock, emergency stop, and testing without blade installed first.                                                                        |
| Blade Motor Controller              | BTN7960B H-Bridge            |      43A rating, 3.3V PWM logic, braking capable | Controls 775 blade motor. Use soft-start and default-off behavior. Arduino should control blade enable, but hardware E-stop should be able to cut blade power independently.                           |
| LiDAR                               | YDLIDAR X2L                  |              360° 2D LiDAR, 8m range, USB serial | ROS2 perception sensor. Should publish `/scan`. Useful for obstacle detection, mapping, and local navigation.                                                                                          |
| Wheel Odometry                      | AS5600 Encoders ×2           |                    12-bit magnetic encoders, I2C | Wheel position/speed feedback. Can be read by Arduino or Pi. For real-time motor feedback, Arduino is probably better; then publish odometry to ROS2 over serial.                                      |
| I2C Expansion                       | TCA9548A Multiplexer         |                                  I2C multiplexer | Allows multiple AS5600 encoders with same/default I2C address. Can also isolate I2C devices if needed.                                                                                                 |
| IMU                                 | ICM-20948                    |                   9-axis accel/gyro/magnetometer | Orientation and heading correction. Should publish `/imu/data`. Magnetometer may be noisy near motors, so gyro/accel may be more useful than compass indoors/near current wiring.                      |
| GPS                                 | ATGM336H GPS                 |                     UART output, ceramic antenna | Outdoor positioning and geofence support. Good for coarse global location, but should not be the only safety boundary for blade operation.                                                             |
| Communication Link                  | USB-C cable, Pi to Uno R4    |                Arduino appears as `/dev/ttyACM0` | Serial control link between ROS2 node and Arduino firmware. Already tested with `LED_ON` / `LED_OFF`.                                                                                                  |
| Software Bridge                     | `mower_serial` ROS2 package  |                    Python `rclpy`, serial bridge | ROS2 node that converts topics into serial commands for the Arduino. Currently working as `/mower/arduino_cmd` string bridge.                                                                          |
| Safety Protocol                     | Arduino watchdog / heartbeat |                  Serial heartbeat from Pi to Uno | Critical safety layer. Arduino should stop drive and blade if heartbeat times out, serial disconnects, E-stop triggers, or invalid command is received.                                                |
